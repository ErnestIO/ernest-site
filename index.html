Define, Manage and Collaborate on your infrastructure

Ernest allows you to create, manage and collaborate on your infrastructure, and do it through an easy single config file. Ernest is an open source layer on top of the most common cloud platform APIs, so you can interact with all of them through a simple yaml file.

REPRODUCIBLE MULTI-PLATFORM INFRASTRUCTURE

A single and easy to understand YAML file to manage your whole infrastructure.

  • Single workflow

  • Reproducible environments

CENTRALIZED TOOL MEANT FOR COLLABORATION

  • Centralized definition store

    Ernest takes care of your definitions on a centralized and auth based platform.

  • Avoid conflicts

    Ernest prevents synchronization issues when two definitions are applied at the same time.

  • Always up to date

    Ever forgot to commit your changes? Ernest will know exactly what's your last infrastructure definition.

EXAMPLES

AWS Basic example

Basic AWS example creating an instance....

---
name: demo
datacenter: my-dc
vpc_id: vpc-abcdef01

networks:
  - name: public
    subnet: 10.0.10.0/24
    public: true

security_groups:
  - name: public-sg
    egress:
      - ip: 0.0.0.0/0
        protocol: any
        from_port: '0'
        to_port: '65535'
    ingress:
      - ip: 52.211.19.211/32
        protocol: tcp
        from_port: '22'
        to_port: '22'

instances:
  - name: public
    elastic_ip: true
    type: t2.micro
    image: ami-ed82e39e
    network: public
    start_ip: 10.0.10.11
    count: 1
    key_pair: my-key
    security_groups:
      - public-sg
